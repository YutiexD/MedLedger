# Phase 6: Plan 1 — Admin Dashboard & On-chain Role Management

<objective>
Implement a secure admin dashboard where the hospital administrator can manage users and synchronize doctor roles on the Sepolia blockchain.
</objective>

<tasks>
<task>
  <description>Create Admin Users List API</description>
  <read_first>
    - src/models/User.ts
  </read_first>
  <action>
    Create `src/app/api/admin/users/route.ts`.
    Implement `GET` logic that:
    1. Authenticates the admin (check `role: "ADMIN"`).
    2. Returns a list of all `User` documents excluding passwords.
  </action>
  <acceptance_criteria>
    - `/api/admin/users` accurately returns the list of registered patients and doctors.
  </acceptance_criteria>
</task>

<task>
  <description>Implement On-chain Role Management Actions</description>
  <read_first>
    - contracts/HospitalMedLedger.sol
    - src/middleware/blockchain-auth.ts
  </read_first>
  <action>
    In `src/app/admin/page.tsx`, implement handlers that use `ethers.js` to:
    1. Grant `DOCTOR_ROLE`: `grantRole(DOCTOR_ROLE, walletAddress)` in the contract.
    2. Revoke `DOCTOR_ROLE`: `revokeRole(DOCTOR_ROLE, walletAddress)` in the contract.
    Both should be executed using the Admin’s MetaMask connection.
  </action>
  <acceptance_criteria>
    - Admin can move users from "PATIENT" to "DOCTOR" (and vice-versa) with corresponding on-chain results.
  </acceptance_criteria>
</task>

<task>
  <description>Develop Admin Dashboard UI</description>
  <read_first>
    - src/app/dashboard/page.tsx
  </read_first>
  <action>
    Create `src/app/admin/page.tsx`.
    Show a tabular view of all users with search and filter support.
    Include "Grant Doctor Role" and "Revoke Role" buttons on each user row where applicable.
    Provide a "New User" modal for manually adding hospital staff.
  </action>
  <acceptance_criteria>
    - Admin dashboard provides clear feedback for role changes and manual additions.
  </acceptance_criteria>
</task>
</tasks>
