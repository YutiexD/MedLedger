# Phase 1: Plan 2 — MongoDB Connection & Smart Contract Init

<objective>
Initialize the MongoDB connection module and configure the Hardhat environment with the target Sepolia ETH smart contract containing RBAC for the Doctor role (`BOOK-01`).
</objective>

<tasks>
<task>
  <description>Setup MongoDB Connection</description>
  <read_first>
    - package.json
  </read_first>
  <action>
    Install `mongoose` and `mongodb`.
    Create `src/lib/mongodb.ts` containing the connection caching logic for Mongoose in Next.js.
    Ensure `process.env.MONGODB_URI` is correctly loaded. 
  </action>
  <acceptance_criteria>
    - `src/lib/mongodb.ts` contains an exported `connectToDatabase` or similar connection function.
    - `package.json` contains `mongoose`.
  </acceptance_criteria>
</task>

<task>
  <description>Setup Hardhat & Core Booking Smart Contract</description>
  <read_first>
    - package.json
  </read_first>
  <action>
    Install Hardhat and OpenZeppelin contracts: `npm install --save-dev hardhat @nomicfoundation/hardhat-toolbox` and `npm install @openzeppelin/contracts`.
    Run `npx hardhat init` (select empty hardhat.config.ts setup). Setup `hardhat.config.ts` supporting the Sepolia testnet based on `process.env.SEPOLIA_RPC_URL` and `process.env.PRIVATE_KEY`.
    Create `contracts/HospitalMedLedger.sol` using Solidity `^0.8.20`.
    Import `@openzeppelin/contracts/access/AccessControl.sol`.
    Define `bytes32 public constant DOCTOR_ROLE = keccak256("DOCTOR_ROLE");`.
    In the constructor, grant `DEFAULT_ADMIN_ROLE` to `msg.sender`.
    This satisfies `BOOK-01` (Admin can assign/revoke Doctor roles using Role-Based Access Control on-chain).
  </action>
  <acceptance_criteria>
    - `contracts/HospitalMedLedger.sol` exists and inherits `AccessControl`.
    - `hardhat.config.ts` has a configuration block for `sepolia`.
    - Smart contract defines `DOCTOR_ROLE`.
  </acceptance_criteria>
</task>
</tasks>
