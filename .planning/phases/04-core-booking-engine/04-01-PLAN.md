# Phase 4: Plan 1 — Smart Contract Transactional Logic

<objective>
Update the `HospitalMedLedger` smart contract to handle booking and cancellation logic with an on-chain mapping to satisfy BOOK-04 and BOOK-05.
</objective>

<tasks>
<task>
  <description>Update HospitalMedLedger.sol</description>
  <read_first>
    - contracts/HospitalMedLedger.sol
  </read_first>
  <action>
    Add `mapping(uint256 => address) public bookings;` to the contract.
    Implement `bookSlot(uint256 slotId)`:
    - Check that `bookings[slotId] == address(0)`.
    - Set `bookings[slotId] = msg.sender`.
    - Emit `SlotBooked(uint256 slotId, address patient)`.
    Implement `cancelSlot(uint256 slotId)`:
    - Check that `bookings[slotId] == msg.sender` (only the patient or an admin/doctor if logic added).
    - Set `bookings[slotId] = address(0)`.
    - Emit `SlotCancelled(uint256 slotId)`.
  </action>
  <acceptance_criteria>
    - `contracts/HospitalMedLedger.sol` contains the `bookings` mapping and the logic for booking/cancellation.
    - Contract compiles successfully.
  </acceptance_criteria>
</task>
